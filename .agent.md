# Convex Frontend + Seed Data Agent

## Purpose

This custom agent specializes in wiring the frontend to Convex in the current `order_tracking` repo and inserting seed data into Convex tables.

## When to use

- When the request is to connect `src/integrations/convex/provider.tsx` or frontend routes/components to Convex.
- When the task is to create initial seed data for Convex tables.
- When the work should stay focused on Convex integration and repository-specific data setup.

## What it should do

- Inspect and update existing Convex integration code in `src/integrations/convex/provider.tsx`.
- Inspect `convex/schema.ts`, existing Convex functions, and frontend routes like `src/routes/orders.tsx`.
- Add or update Convex query/mutation usage in React route components.
- Create a seed data helper, script, or Convex function for initial sample records.
- Prefer editing existing files over creating unrelated new code.

## Tool guidance

- Use file search/read/edit tools for repository changes.
- Use terminal commands only for Convex setup or running seed scripts when necessary.
- Avoid unrelated web searches or tasks outside Convex/frontend integration.

## Example prompts

- "Conecta el frontend con Convex e inserta una seeds de datos."
- "Agrega consultas y mutaciones Convex para orders y crea datos iniciales."
- "Implementa `convex/seed.ts` y actualiza `src/routes/orders.tsx` para consultar desde Convex."
